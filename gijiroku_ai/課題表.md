# 議事録Bot 課題表

## システム概要
- LINE公式アカウントに音声ファイルを送信すると、AIが自動で議事録を生成して返信するシステム
- 使用技術：Python / FastAPI / LINE Messaging API / Google Gemini API / Google Cloud Run

---

## 現状の仕様

| 項目 | 内容 |
|---|---|
| 対応ファイル形式 | m4a / mp3 / その他音声ファイル |
| ファイルサイズ上限 | 200MB以内 |
| 処理時間目安 | 1〜2分（音声の長さによる） |
| 出力先 | LINEトーク画面への返信 |
| 対応言語 | 主に日本語 |
| サーバー | Google Cloud Run（東京リージョン） |

---

## 現状の課題

| No | 課題 | 優先度 | 難易度 |
|---|---|---|---|
| 1 | 話者識別未対応（誰が話したか不明） | 高 | 中 |
| 2 | 処理中にエラーが起きても再試行できない | 高 | 低 |
| 3 | 議事録のフォーマットが固定（カスタマイズ不可） | 中 | 低 |
| 4 | 長時間音声（1時間超）の処理精度が下がる場合がある | 中 | 高 |
| 5 | 音声品質が悪い場合（雑音・複数人同時発話）の精度低下 | 中 | 高 |
| 6 | 処理状況の進捗が「作成中...」のみで詳細不明 | 低 | 低 |
| 7 | ファイル以外の形式（URL・テキスト）には未対応 | 低 | 中 |
